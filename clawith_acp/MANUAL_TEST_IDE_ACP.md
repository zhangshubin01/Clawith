## 0. 环境与数据准备


| 项         | 说明                                                                                                             |
| --------- | -------------------------------------------------------------------------------------------------------------- |
| 后端        | Clawith 已启动，`clawith-acp` 插件已加载；能访问 `CLAWITH_URL`。                                                             |
| 瘦客户端      | `.env` 已配置 `CLAWITH_URL`、`CLAWITH_API_KEY`、`CLAWITH_DEFAULT_AGENT_ID`；按 README / `setup-*.sh` 装好依赖与 `.venv`。 |
| JetBrains | `~/.jetbrains/acp.json`（或 Windows 等价路径）中 **command** 指向本机 `.venv` 的 `python`，**args** 指向本目录 `server.py`（绝对路径）。 |

| 模型 B      | 另备一个 **仅文本** 模型或同一智能体临时改为 `supports_vision = false`（用于「不支持视觉」用例）。                                              |


**建议记录**：后端日志中 ACP WS 连接、每次 `prompt` 是否有报错；IDE 侧「Get ACP Logs」如有可一并对照。

## 6. 取消与会话并发（WebSocket `schemaVersion: 3`）

**说明**：云端已接 **`session/cancel` → WS `cancel`**、**跨连接 cancel 注册表**、结束帧 **`cancelled`（与 `done` 分离）**；`call_llm` 会向 LLM `stream` 传入 `cancel_event` 以尽早打断流式读。瘦客户端对云端 `cancelled` 会映射 **`PromptResponse.stopReason = cancelled`**。

### TC-11 IDE 内取消（同一条 prompt 连接）


| 步骤  | 操作                                                                 |
| --- | ------------------------------------------------------------------ |
| 1   | 发起一条**较长**的生成（大段文本或易触发多轮工具的问题）。                                      |
| 2   | 在仍在生成时触发 IDE 的 **Cancel / 停止**（走 ACP `session/cancel` → 瘦客户端向云端发 `cancel`）。 |



| 期望                                                                                                                                      |
| --------------------------------------------------------------------------------------------------------------------------------------- |
| 生成在合理时间内停止；**不应**再收到本轮的 `done` 作为成功结束（应走 **`cancelled`**）。IDE 侧可见「已取消」类提示（瘦客户端对 `cancelled` 的 `session_update`）。 |
| 助手轮历史里可能出现 `[Cancelled]` 文本，属预期。                                                                                                        |


---

### TC-12 第二条 WebSocket 发 cancel（跨连接）


| 步骤  | 操作                                                                                                                                                                                                                           |
| --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 连接 A：对某 `SESSION_ID` 发 `prompt` 并保持生成中。                                                                                                                                                                                    |
| 2   | 连接 B：**同一** `agent_id`、**同一** `token`（同一用户），另开一条 WS，仅发送：`{"schemaVersion":3,"type":"cancel","session_id":"<SESSION_ID>"}`（可用 `websocat` 或自写脚本；勿与连接 A 同进程阻塞，建议脚本/第二终端）。 |



| 期望                                                |
| ------------------------------------------------- |
| 连接 A 上在途 prompt 被取消，现象与 **TC-11** 一致（收到 `cancelled` 等）。 |


---

### TC-13 同 session 并发第二个 prompt（应拒绝）


| 步骤  | 操作                                                                                              |
| --- | ----------------------------------------------------------------------------------------------- |
| 1   | 在**同一 WebSocket 连接**上，对同一 `session_id`：**第一条 `prompt` 尚未结束**时，立即再发第二条 `prompt`（自动化脚本或极快双击发送；IDE 通常串行，可能需脚本复现）。 |



| 期望                                                                                |
| --------------------------------------------------------------------------------- |
| 第二条应收到云端 **`type: error`**，文案含「已有 prompt 在进行」之意；不应把两轮用户消息都当成成功开跑。第一条仍按原逻辑结束（`done` / `cancelled` / `error`）。 |


---

### TC-14 敏感 ide_* 与权限（可选）


| 步骤  | 操作                                                                                    |

| ------------------------------------------------------------------------------------- |
| 拒绝时云端工具结果为拒绝类说明；允许时实际写入/执行符合预期。环境变量 `allow` / `deny` 可覆盖 IDE 对话框（用于 CI/自动化）。 |

## 7. 已知限制（测到不必报 bug）


| 现象        | 说明                                                                                    |
| --------- | ------------------------------------------------------------------------------------- |
| 重连后图片历史丢失 | 多模态 **落库为摘要**（如 `[图片 1]`），DB 水合不会还原 base64。                                            |
| `cancel` 语义  | 云端 `cancel` **必须**带 `session_id`；缺省时后端仅打日志，不会广播取消。OpenAI **Responses** 协议路径对流式中途打断能力弱于原生 SSE。 |
| `audio` 块   | 瘦客户端当前 **不上传** 到云端。                                                                   |

## 9. 附录：用 `websocat` / 脚本粗测 WS（可选）

不经过 IDE，仅验证后端对 `prompt_parts` 的解析（需有效 `agent_id` + `token`）：

```bash
# 示例：请替换 HOST、AGENT_ID、TOKEN、SESSION_ID（32 位 hex）
echo '{"schemaVersion":3,"type":"prompt","session_id":"'"$SESSION_ID"'","text":"ping"}' \
  | websocat -t "wss://HOST/api/plugins/clawith-acp/ws?agent_id=AGENT_ID&token=TOKEN"
```

带图片时把 `text` 换成 `prompt_parts` JSON（与瘦客户端发送结构一致）。若环境无 `websocat`，可跳过本附录。

**跨连接 cancel 示例**（另开终端，与正在跑 `prompt` 的 `SESSION_ID` 一致）：

```bash
echo '{"schemaVersion":3,"type":"cancel","session_id":"'"$SESSION_ID"'"}' \
  | websocat -t "wss://HOST/api/plugins/clawith-acp/ws?agent_id=AGENT_ID&token=TOKEN"
```
